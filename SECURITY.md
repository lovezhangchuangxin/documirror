# Security Policy

## Reporting a vulnerability

Please do not open public issues for suspected security problems.

Send reports to `2911331070@qq.com` with:

- a description of the issue
- affected commands, inputs, or configuration
- reproduction steps or a proof of concept
- impact assessment if known

We will acknowledge receipt, investigate, and coordinate a fix before public
disclosure when appropriate.

## Scope

Security reports are most helpful when they focus on vulnerabilities in:

- CLI command execution
- repository scaffolding
- file handling and output generation
- credential handling in `.env`
- network requests and remote content processing
